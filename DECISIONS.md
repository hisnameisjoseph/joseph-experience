# Decisions

- 2026-07-19: Pinned @react-three/fiber v8 and @react-three/drei v9 (not latest v9/v10) because the newer majors require React 19 and the spec mandates React 18.
- 2026-07-20: Pivoted to a 2D canvas game per revised spec: removed three/@react-three/fiber/@react-three/drei entirely; game engine will be hand-written Canvas 2D in src/game/ with no rendering library.
- 2026-07-20: Modeled RoomObject as a discriminated union (kind: 'card' | 'door') so cardId/doorTarget are required exactly when applicable, instead of optional fields on one interface.
- 2026-07-20: Canvas scaling done in CSS (fixed 400x225 backing store, CSS size = largest integer multiple, image-rendering: pixelated) rather than resizing the backing store — game code always draws in one coordinate space.
- 2026-07-20: Keyboard input keys matched by physical position (KeyboardEvent.code) so WASD works on non-QWERTY layouts; on diagonal movement the current facing is kept while still held, so facing never flickers.
- 2026-07-20: Collision is axis-separated AABB vs the tile grid: x resolves before y so walls slide naturally; movement advances in at-most-one-tile substeps and snaps flush to the blocking tile (no tunneling at any delta); box edges are half-open so standing flush against a wall is legal; tiles outside the grid count as solid.
- 2026-07-20: Interaction targeting = the grid cell one step from the player-center's cell in the facing direction (pure findInteractable); E is edge-triggered via a consumed queue so holding E fires once, and presses made while a card is open are drained, never replayed on close.
- 2026-07-20: Room transitions are a pure state machine (src/game/transition.ts): fade-out accumulates a black overlay alpha from delta time, the room swap happens exactly once at full black (player placed at the target room's spawn, store updated), then fade-in; the world is frozen and E presses are drained during the whole fade, so spamming E cannot chain transitions. Doors to rooms that don't exist yet are inert (no plaque, no-op E).
- 2026-07-20: Touch controls feed the same polled GameInput as the keyboard (directions OR-merged, action button queues the same edge-triggered interact), so game logic is input-agnostic; visibility is pure CSS via (pointer: coarse), and pointer capture keeps a held D-pad button delivering its release even if the thumb slides off.
- 2026-07-22: Per-room palette added as RoomData.palette (tileId -> color) overriding the default tile colors, so each room themes the same tile ids (Focus Bear = dark blues) without new tile ids per room; walkability stays global per tile id.
- 2026-07-22: Interactable objects gained an optional sprite field (clocks/terminal/robot/cabinet), drawn by the render layer from data; each prop sits on a solid tile (wall or TILE_PROP) so the player stops adjacent and faces it. Render layer split into render.ts + sprites.ts to keep files under ~150 lines; game logic still never touches the canvas.
- 2026-07-22: Hub dressed as a home office via RoomData.palette (warm wood/gold) plus a rug tile and a laptop sprite on the desk. All five doors always show their plaque when faced; doors to unbuilt rooms trigger a transient centered "Coming soon" toast (timed in the loop, drawn by the render layer) instead of a transition. Plaque font bumped to 8px with a bordered box for readability at internal resolution.
